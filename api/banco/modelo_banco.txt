1. Módulo de Identidade e Acessos (IAM)

A segurança e o controle de acesso exigem proteção rigorosa de dados sensíveis e permissões granulares.

• usuarios (cadastro central)
Campos: id (UUID), nome, email, senha_hash, cpf, id_nivel_acesso, data_criacao
→ Senhas obrigatoriamente com hash Argon2id.

• usuarios_enderecos (endereços múltiplos)
Campos: id, id_usuario, titulo, cep, logradouro, numero, bairro, cidade, padrao_entrega
→ Ex: Casa, Escritório, Hotel.

• niveis_acesso (papéis do sistema)
Campos: id, nome, permissoes_json
→ Ex: Admin Global, Logística Galpão, Cliente VIP.

• api_keys (integrações e dispositivos)
Campos: id, chave_hash, descricao, data_expiracao, status_ativo
→ Ex: leitor RFID enviando requisições.

2. Módulo de Ativos de Luxo (O “Cofre”)

Gerencia a riqueza visual e precisão técnica dos itens.

• produto_midia (assets visuais)
Campos: id, id_produto, tipo_midia, url_arquivo, ordem_exibicao
→ Tipos: imagem 4K, vídeo, modelos 3D (.glb/.gltf).

• atributos_medidas (medidas reais)
Campos: id, id_item_estoque, ombro_cm, busto_cm, cintura_cm, quadril_cm, comprimento_cm
→ Reduz erros de caimento.

3. Módulo Logístico e Operacional (O “Fluxo”)

Espelha todo o ciclo físico da peça.

• locacoes (contrato de aluguel)
Campos: id, id_usuario, id_item_estoque, data_inicio, data_fim, valor_aluguel, valor_caucao, status_pedido

• pontos_coleta (hubs físicos)
Campos: id, nome_unidade, cep, logradouro, horario_funcionamento
→ Ex: QG Arqon Jardins.

• entregas (envio da peça)
Campos: id, id_locacao, id_entregador, status_rota, codigo_rastreio
→ Integração com APIs de frete.

• devolucoes (retorno da peça)
Campos: id, id_locacao, id_ponto_coleta, data_recebimento, status_triagem
→ Status: devolvido → aguardando vistoria.

4. Módulo Financeiro e Revenue Share

Controle financeiro completo e rastreável.

• transacoes_financeiras
Campos: id, id_locacao, tipo, status, gateway_id, valor_total
→ Integração com Stripe ou Pagar.me.

• caucoes
Campos: id, id_locacao, valor_retido, status_caucao, data_captura_ou_estorno

• split_pagamentos
Campos: id, id_transacao, id_marca, valor_marca, valor_arqon, status_repasse

• wishlist

14. Logs de Integração (Webhooks)

Monitoramento de integrações externas.

• logs_webhooks
Campos: id, servico_origem, payload_recebido, status_processamento, erro_mensagem
→ Armazena JSON de Stripe, Correios, RFID.
